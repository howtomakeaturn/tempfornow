Atayal is a CMS composed of purely some Laravel Packages.

Why?

Because Laravel itself is flexible enough, we shouldn't make more magics on top of it.

Just utilize what Laravel provides.

To extend Atayal should be just to drag and drop your Laravel Packages to 'Atayal/Modules' folder.

No magics at all?

Atayal still has few magics, otherwise it's not a CMS but just Laravel anyway.

Module Strucure

Atayal needs you to create your Module under 'Atayal/Modules' folder.

Folder naming

Atayal needs you to create a Main folder as Laravel package under your Module.

If you also want to create an admin panel for your Module, create an Admin folder as Laravel package under the same Module.

Atayal will load your packages automatically. And that's what all Atayal do for you.

Namespaces

Because all Modules classes (including those made by you) are loaded by Composer under PSR-0, you need to obey it, too.

For example, the Page Module are under namespace Atayal\Modules\Page.
